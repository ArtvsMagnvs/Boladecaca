2. ERRORES PENDIENTES PARA CLAUDE CODE
Los siguientes problemas fueron detectados en la auditoría pero NO están corregidos todavía. Claude Code debe aplicarlos después de la configuración de MiniMax.
Pendiente #1 — AgentResponse schema incompleto
Severidad: MEDIA. El schema AgentResponse en backend/app/db/schemas.py solo devuelve id, name, system_prompt y created_at, pero el modelo de base de datos Agent tiene también agent_type, description e is_active. La pantalla de Agentes no puede mostrar ni editar estos campos.
Acción requerida en backend/app/db/schemas.py:
Buscar la clase AgentResponse y añadir los campos faltantes:
class AgentResponse(BaseModel):
    id: int
    name: str
    agent_type: Optional[str] = None    # AÑADIR
    description: Optional[str] = None   # AÑADIR
    system_prompt: Optional[str] = None
    is_active: bool = True               # AÑADIR
    created_at: Optional[datetime] = None
    class Config:
        from_attributes = True
También actualizar AgentCreate para aceptar los mismos campos:
class AgentCreate(BaseModel):
    name: str
    agent_type: Optional[str] = None    # AÑADIR
    description: Optional[str] = None   # AÑADIR
    system_prompt: Optional[str] = None
    is_active: bool = True               # AÑADIR
Pendiente #2 — Voice status: mismatch entre backend y frontend
Severidad: MEDIA. El endpoint GET /api/voice/status devuelve un objeto anidado {elevenlabs: {...}, espeak: {...}} pero api.ts declara que devuelve {configured: boolean, voices_count: number, message: string}. La pantalla de VoiceCenter puede fallar al leer el estado.
Acción requerida en frontend/src/lib/api.ts:
Buscar la interfaz de getVoiceStatus y actualizar el tipo de retorno:
async getVoiceStatus(): Promise<{
  elevenlabs: { configured: boolean; voices_count: number; message: string };
  espeak: { available: boolean; voices_count: number; message: string };
  fallback: string;
  recommended: string;
}>
Pendiente #3 — .gitignore no incluye backend/.env
Severidad: ALTA (seguridad). El archivo backend/.env con la API key de MiniMax no debe subirse a ningún repositorio. Claude Code debe verificar que .gitignore en la raíz del proyecto incluye la línea:
backend/.env
.env
Si no están, añadirlas al archivo .gitignore existente.
Pendiente #4 — Eliminar aithera.db huérfana del repositorio
Severidad: BAJA. Existe el archivo backend/aithera.db en el repositorio que NO es la base de datos activa (la real está en %APPDATA%/Aithera/aithera.db). Claude Code debe eliminar este archivo con git rm --cached backend/aithera.db y añadir *.db al .gitignore para evitar confusión futura.
