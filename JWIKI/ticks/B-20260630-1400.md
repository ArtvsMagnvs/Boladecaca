## Por qué el spawn falló — diagnóstico técnico (para próximo tick)

El error "Invalid spawn config JSON: {" sugiere que el daemon `mavis.cmd` pierde las comillas dobles en el payload al pasar por cmd.exe → node.js. Las variantes probadas:

- `mavis communication send --content "...JSON..."` desde PowerShell → args shell-escaped → `"` se pierden.
- `python subprocess.run(['mavis', ..., '--content', json])` → similar pérdida.
- `python subprocess.run(shell=True, ...)` → misma pérdida.
- `mavis communication send --content "@filepath"` → esperado file-ref, daemon no expandió.
- Probado escape `^"` (cmd caret) → falló.

**Recomendación para próximo turno**:

1. Pedir al usuario que valide que el formato de spawn funciona en modo interactivo (no en cron) — si sí, hay regresión específica en cron.
2. Considerar alternativa: usar el comando `mavis session new aithera-wiki-inv2` (si existe) en lugar de `--command spawn`, ya que se sabe que da error desde el system prompt.
3. Documentar este workaround en `KNOWN_PITFALLS` cuando se cree el doc (JWIKI-246+).
